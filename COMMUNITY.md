# Community Playbook

How we operate:
- Use Issues for bug reports and feature requests.
- Use labels: good-first-issue, help-wanted, discussion.
- Response time goal: acknowledge new issues within 72 hours.

Maintainers
- See CODEOWNERS for maintainers. PRs reviewed by maintainers.

Communication
- Use GitHub Discussions or Issues for public discussion.
- For private maintainer coordination, use email or a private chat channel.

Releases
- Create GitHub releases with changelogs and upgrade notes.
